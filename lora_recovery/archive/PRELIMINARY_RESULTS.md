# Предварительные проверки

Этот архив содержит технические проверки, выполненные до финального
воспроизводимого sweep с адаптерами. Они сохранены только как история
реализации и не являются источником финального CSV, графиков, метрик или
выводов. Финальный отчёт: `../LORA_RESULTS.md`.

## Архивированные конфигурации

- `configs/smoke_xpu_lora.yaml`: конфигурация smoke-теста XPU/LoRA. Она
  использовала `Qwen/Qwen3-0.6B-Base`, три примера и предназначалась только
  для проверки исходного технического пути.
- `configs/validation_xpu_qwen3.5_0.8b.yaml`: online-KD validation LoRA для
  Qwen3.5-0.8B на 64 примерах.
- `configs/recovery_xpu_qwen3.5_0.8b_a005.yaml`: online-KD recovery-проверка
  на 64 примерах при `alpha=0.05`.
- `configs/pilot_synthetic_ce_xpu_qwen3.5_0.8b.yaml`: технический pilot
  автономного `synthetic_ce` на 120 шагах; это не был полный проход по 1 500
  строкам.

## Зафиксированные наблюдения

- Intel XPU был доступен через `torch.xpu`; загрузка BF16-модели, forward,
  backward и настоящий LoRA optimizer step завершились без NaN/Inf.
- Online-KD проверки на 64 примерах подтвердили работоспособность путей метрик
  и LoRA, но использовали teacher в training loss и поэтому не сопоставимы с
  финальным автономным objective.
- 120-step pilot `synthetic_ce` использовал Qwen3.5-0.8B, XPU BF16, LoRA rank
  8, `alpha=0.05`, batch size 1, LR `5e-5` и warmup 10. PPL улучшился с
  28.3967 до 25.6201; KL T→S изменился с 0.1459 до 0.1633, KL S→T — с 0.1619
  до 0.1713, CKA — с 0.9850 до 0.9844. `teacher_train_forwards=0`, но прогон
  состоял только из 120 шагов, а не полного epoch.

Отдельные preliminary raw-каталоги результатов в рабочем дереве не сохранены.
Восемь финальных raw-прогонов намеренно находятся вне этого архива и являются
единственным источником финальных значений.
