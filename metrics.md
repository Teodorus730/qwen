# 📊 Метрики Pretrain-оценки: математика и чек-лист запуска

### [Репозиторий](https://github.com/EleutherAI/lm-evaluation-harness)

**Краткий итог:** Каждая метрика - это способность модельки предсказывать след слово и насколько уверенно она это делает (энтропия по распределению токенов должна быть маленькая). Дальше берем несколько датасетов (общий, математика, рассуждения, тесты и прочее) и считаем accuraccy по этим метрикам для нашей модельки и для [qwen3.5-0.8B-Base](https://huggingface.co/Qwen/Qwen3.5-0.8B-Base) - сравниваем получившееся и радуемся или огорчаемся

## 🔢 1. Математическая суть метрик

| Метрика                               | Формула                                                                                                                                       | Что измеряет                                                                                                        |
| :------------------------------------ | :-------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------ |
| **Log-Likelihood**                    | $\displaystyle \log P(W) = \sum_{i=1}^{N} \log P(w_i \mid w_{<i})$                                                                            | Суммарная уверенность модели в реальных токенах тестового текста                                                    |
| **Cross-Entropy Loss**                | $\displaystyle \mathcal{L}_{CE} = -\frac{1}{N} \sum_{i=1}^{N} \log P(w_i \mid w_{<i})$                                                        | Средняя отрицательная лог-вероятность (целевая функция при обучении)                                                |
| **Perplexity (PPL)**                  | $\displaystyle \text{PPL} = \exp(\mathcal{L}_{CE}) = \exp\left(-\frac{1}{N}\log P(W)\right)$                                                  | «Эффективное число вариантов» на каждом шаге. Чем ближе к `1`, тем лучше                                            |
| **Few-Shot Accuracy** (MC)            | $\displaystyle \text{Acc} = \frac{1}{M}\sum_{j=1}^{M} \mathbb{I}\!\left(\arg\max_{c\in\mathcal{C}} \log P(c\mid\text{prompt}_j) = y_j\right)$ | Доля задач, где модель выбирает правильный вариант по максимальной условной вероятности                             |
| **Pass@k** (код/генерация)            | $\displaystyle \text{Pass@k} = \mathbb{E}\!\left[1 - \frac{\binom{n-c}{k}}{\binom{n}{k}}\right]$                                              | Вероятность, что среди `k` сгенерированных ответов найдётся ≥1 корректный (`n` — всего сгенерировано, `c` — верных) |
| **KL-Divergence** (сравнение выходов) | $\displaystyle D_{KL}(P_{\text{ref}} \| P_{\text{my}}) = \sum_x P_{\text{ref}}(x) \log \frac{P_{\text{ref}}(x)}{P_{\text{my}}(x)}$            | Насколько распределение вероятностей твоей модели расходится с референсом на тех же промптах                        |

> ⚠️ **Важно:** `lm-eval` оценивает **функциональное поведение** (качество предсказаний/решений). Сравнивать напрямую вектора весов (`L2`, косинус) **бессмысленно** из-за симметрий в архитектуре и стохастичности оптимизации.

## 🚀 2. Чек-лист запуска через `lm-evaluation-harness`

### ✅ Подготовка

- [ ] Установить: `pip install "lm_eval[hf,vllm]"`
- [ ] В папке модели обязаны лежать: `config.json`, `pytorch_model.bin`/`model.safetensors`, `tokenizer.json`, `vocab.json`
- [ ] Убедиться, что `modeling_qwen.py` присутствует или указан `trust_remote_code=True`

### ▶️ Команда запуска

```bash
lm-eval run \
  --model hf \
  --model_args pretrained=./my_model,tokenizer=./my_model,dtype=bfloat16,trust_remote_code=True \
  --tasks mmlu,gsm8k,evalplus_humaneval \
  --num_fewshot 5 \
  --batch_size auto \
  --device cuda:0 \
  --output_path ./results
```

### ⚙️ Критические флаги

| Параметр                            | Значение            | Почему                                                    |
| :---------------------------------- | :------------------ | :-------------------------------------------------------- |
| `trust_remote_code=True`            | Обязательно         | Qwen использует кастомные слои/токенизатор                |
| `temperature=0` / `do_sample=False` | По умолчанию для MC | Исключает стохастичность, оценка идёт по `argmax` логитов |
| `max_length=2048`                   | Рекомендуется       | Бенчмарки не требуют 32k, экономит VRAM в ~5-10 раз       |
| `--limit 10`                        | Для отладки         | Проверь пайплайн перед полным прогоном                    |

### 📐 Правила сравнения с `Qwen3.5-0.8B-Base`

- [ ] Запускать **идентичные версии бенчмарков** (одинаковые few-shot примеры, промпт-шаблоны)
- [ ] Считать PPL на **независимом валидационном срезе** (не участвовал в обучении!)
- [ ] Разница в Accuracy ≤ `±1.5%` считается статистически незначимой для `0.8B`
- [ ] Для оценки «похожести поведения» посчитай средний `KL-div` выходных распределений на 500 фиксированных промптах: `<0.1` — почти идентичны, `>1.0` — разные стратегии генерации

## 📌 Краткий итог

| Что оцениваем           | Как измеряем                            | Где смотреть                   |
| :---------------------- | :-------------------------------------- | :----------------------------- |
| Сходимость pretrain     | `val_loss` → `PPL = exp(loss)`          | Логи обучения                  |
| Обобщающая способность  | PPL на внешних корпусах (C4, Wikipedia) | `lm-eval` или кастомный скрипт |
| Интеллектуальные навыки | Few-Shot Accuracy (MMLU, GSM8K)         | `lm-eval --tasks ...`          |
| Качество кода           | Pass@1 / Pass@10                        | `lm-eval --tasks evalplus_*`   |
| Близость к референсу    | KL-div распределений + дельта точности  | Кастомный Python-скрипт        |
